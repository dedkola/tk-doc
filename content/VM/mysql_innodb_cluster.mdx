Takes from Official MySQL Operator documentation [here](https://dev.mysql.com/doc/mysql-operator/en/mysql-operator-installation.html)

Install K3s and MetalLB here [K3s with LoadBalancer MetalLB](/docs/vm/k3s_metallb_loadbalancer)

### Deploy MySQL Operator
```bash
kubectl apply -f https://raw.githubusercontent.com/mysql/mysql-operator/trunk/deploy/deploy-crds.yaml
kubectl apply -f https://raw.githubusercontent.com/mysql/mysql-operator/trunk/deploy/deploy-operator.yaml
```

Verify the deployment
```bash
kubectl get deployment mysql-operator --namespace mysql-operator
```
### Create credentials secret

```bash
kubectl create secret generic mypwds \
        --from-literal=rootUser=root \
        --from-literal=rootHost=% \
        --from-literal=rootPassword="123"
```
### Deploy MySQL InnoDB Cluster
```bash
nano mycluster.yaml

```
```yaml
apiVersion: mysql.oracle.com/v2
kind: InnoDBCluster
metadata:
  name: mycluster
spec:
  secretName: mypwds
  tlsUseSelfSigned: true
  instances: 3
  router:
    instances: 1
```

```bash
kubectl apply -f mycluster.yaml
kubectl get innodbcluster --watch
```

### Expose MySQL InnoDB Cluster to LAN

Dont do this!!!

```bash
kubectl patch service mycluster -p '{"spec": {"type": "LoadBalancer"}}'
```

### Connect Mysqlsh
```bash
 kubectl run --rm -it myshell --image=container-registry.oracle.com/mysql/community-operator -- mysqlsh
```
### Connect to specific pod in cluster
```bash
kunectl get pods
kubectl --namespace default exec -it mycluster-2 -- bash

mysqlsh root@localhost
```
